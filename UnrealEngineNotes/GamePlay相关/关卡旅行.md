在unreal中travel（旅行）的意思是加载指定的关卡，并把玩家一起带进这个关卡。

# seamless 和 non-seamless
旅行有两种方式无缝和非无缝。
## 无缝：
- 无缝旅行是异步加载关卡，玩家保持连接，但是会被先转移到异步加载的过渡地图中，直到目标关卡加载完成再一起进入目标关卡。
**无缝旅行的3种发生方式**：
1. 首次加载地图时。（指的是初始连接加载的第一张地图，而不是后续的任何新地图）
2. 首次作为客户端连接到服务器时（如joining session）。
3. 结束多人游戏并开启新的游戏时。
# 非无缝
- 非无缝旅行是同步加载关卡，玩家会和服务器先断开连接，在新地图加载完后重新连接。

# 旅行类型
**无缝**
TRAVEL_Relative：同一服务器，保留最后的选项字符串。当前的url相对于最后的url。

**非无缝**
TRAVEL_Partial：服务器重置，保留最后的的选项字符串。当前的url是最后的url的一部分、
TRAVEL_Absolute：服务器重置，忽略最后的选项字符串。当前url是绝对的，意味着最后一个url（包括最后的选项字符串）被清除。

# 驱动旅行的方式
## C++侧
**UEngine::Browse()**：
- 加载新地图时就像硬重置一样。
- 总会导致非无缝旅行。
- 会导致服务器在前往目标地图之前断开当前客户端的连接。
- 客户端将断开与当前服务器的连接。
- 专用服务器无法迁移到其他服务器，因此地图必须是本地的（不能是 URL）。

**可选择无缝和非无缝**
**UWorld::ServerTravel()**：
- 仅限服务器使用。
- 将服务器跳转到新世界/关卡。
- 所有连接的客户端都将跟随。
- 这是多人游戏在地图间移动的方式，而服务器负责调用这个函数。
- 服务器将调用 `APlayerController::ClientTravel()` ，对所有已连接的客户端玩家。
**APlayerController::ClientTravel()**：
- 如果从客户端调用，将迁移到新的服务器。
- 如果从服务器调用，将指示特定客户端前往新地图（但保持连接到当前服务器）。

## 蓝图驱动旅行的方式
**OpenLevel**：会硬切换关卡即使开了无缝切换。
1. 从客户端调用：将切换到新服务器，而不会断开其他客户端与该服务器的连接。
2. 如果从监听服务器调用：将切换监听服务器玩家到新地图，断开客户端回到入口/默认地图。
3. 如果从专用服务器调用，将导致切换失败，断开客户端回到默认地图，除非切换 URL 具有选项“监听”，这将指示特定客户端作为监听服务器切换到新地图，然而其他客户端将断开回到默认地图。
   对应的函数调动：
   ```cpp
   UGameplayStatics::OpenLevel()
	   UEngine::SetClientTravel()
	   UEngine::TickWorldTravel()
			UEngine::Browse()
			   UEngine::LoadMap() (默认bAbsolute = true)
   ```


**ExecuteConsoleCommand**
此函数和 `Command` 参数可以是 `ServerTravel <MapName>` 或 `Travel <MapName>` （前者通常使用，因为它可以无缝衔接，而后者则不能），它们会前往指定的地图，同时传递之前设置的选项字符串，分别对应相对路径和部分路径的移动。它们分别对应原生函数： `UEngine::HandleServerTravelCommand()` 和 `UEngine::HandleTravelCommand()` 。

另一个你可以使用的命令是 `Open <MapName>` ，它会打开指定的地图，但不传递之前设置的选项字符串，因为它是一次绝对路径的移动。它对应原生函数 `UEngine::HandleOpenCommand()` ，该函数调用 `UEngine::SetClientTravel()` 。在下一个时间点， `UGameEngine::Tick()` 调用 `UEngine::Browse()` ， `UEngine::Browse()` 调用 `UEngine::LoadMap()` 。这始终是一次绝对路径的移动。

# 启用无缝旅行和过渡地图
要启用无缝旅行，需要设置一个过渡地图。这通过 `UGameMapsSettings.TransitionMap` 属性进行配置。默认情况下此属性为空，如果您的游戏不设置此属性，将创建一个空的过渡地图。

过渡地图存在的原因是，必须始终加载一个世界（它包含地图），因此我们不能在加载新地图之前释放旧地图。由于地图可能非常大，同时将旧地图和新地图保存在内存中会是一个坏主意，因此过渡地图应运而生。一旦你设置了过渡地图，就将 `AGameModeBase.bUseSeamlessTravel` 设置为 true，从那时起无缝旅行应该可以工作。

ps：（在 5.1 之前，单进程 PIE（编辑器中运行）不支持无缝旅行。因此，你必须关闭 `Run Under One Process` 设置，或者启动独立的（右键点击你的 .uproject 并 `Launch game` ），或者启动打包构建。自 5.1 版本起，单进程 PIE 支持无缝旅行，但你需要在控制台中启用 CVar `net.AllowPIESeamlessTravel` ，因为它默认是禁用的。）

# Game Options String
游戏选项字符是旅行URL的一部分
```cpp
USTRUCT()
struct ENGINE_API FURL
{
    GENERATED_USTRUCT_BODY()

    // Protocol, i.e. "unreal" or "http".
    UPROPERTY()
    FString Protocol;

    // Optional hostname, i.e. "204.157.115.40" or "unreal.epicgames.com", blank if local.
    UPROPERTY()
    FString Host;

    // Optional host port.
    UPROPERTY()
    int32 Port;

    UPROPERTY()
    int32 Valid;

    // Map name, i.e. "SkyCity", default is "Entry".
    UPROPERTY()
    FString Map;

    // Optional place to download Map if client does not possess it
    UPROPERTY()
    FString RedirectURL;

    // Options.
    UPROPERTY()
    TArray<FString> Op;

    // Portal to enter through, default is "".
    UPROPERTY()
    FString Portal;

    // Statics.
    static FUrlConfig UrlConfig;
    static bool bDefaultsInitialized;

    ...
};
```

**URL** 可以传送到可执行文件中，以强制游戏在启动时载入特定地图。这些也可以与 SERVER 或 EDITOR 模式结合使用，以运行具有特定地图的编辑器或服务器。传入 **URL** 是可选的，但必须立即跟随可执行文件名或任何模式切换（如果存在的话）。

**URL** 由**两部分**组成： **地图名称或服务器 IP 地址**和一系列**可选的附加参数** 。地图名称（`FURL.Map`）可以是位于地图目录中的任何地图。在这里包含文件扩展名（`.umap`）是可选的。要从地图目录中加载未找到的地图，可以使用绝对路径或从地图目录的相对路径。在这种情况下，必须包含文件扩展名。服务器 IP 地址（`FURL.Host` 是一个标准的四部分 IP 地址，由 0 到 255 之间的四个值组成，中间用点隔开。附加选项（`FURL.Op`）通过附加到地图名称或服务器 IP 地址来指定。每个选项前面都用 `?`（作为分隔符），可以使用 `=` 设置值，即： `?option1=value1?option2=value2` . 如果以 `-` 开头的选项，将从缓存的 **URL** 选项中删除该选项。


## 内置url选项
每条连接有两个缓存的 **URL**：
1. **World URL** (`UWorld.URL`): Cached loading world URL.  
    **世界 URL**（`UWorld.URL`）：缓存的加载世界 URL。
2. **Demo URL** (`FReplayHelper.DemoURL`): Cached replay URL.  
    **演示 URL**（`FReplayHelper.DemoURL`）：缓存的重播 URL。
