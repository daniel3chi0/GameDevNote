# 左值和右值基本概念
能够访问到地址的是左值，访问不到地址的是右值。
比如下述代码变量a的地址能在下一行被访问到，所以a是个左值。

```cpp
int a = 1;
cout << &a << endl;
```

但是我们这样写就不行，首先在编译器层面上就不能通过编译。
会有红线报错：**E0158	表达式必须为左值或函数指示符**
这个数字1就是个右值。
```cpp
cout << &1 << endl;
```

# 左值引用和右值引用
类型 + & 是左值引用
```cpp
void Test(int& a)
{
	cout << "Left value" << endl;
}
```

类型 + &&是右值引用
```cpp
void Test(int&& a)
{
	cout << "Right value" << endl;
}
```

可根据参数类型自动调用对应函数
```cpp
int main()
{
	int a = 1;
	Test(a);
	Test(1);
	return 0;
}
```

输出：
Left value
Right value

# 引用折叠
当我们定义模板类参数是T + && 或是T + &时
当T被推导成引用类型时：int& 或 int&& 此时就会发生引用折叠
前面的情况
## 万能引用
前面的情况被称为万能引用
参数类型判断规则使用下面的引用折叠规则
```cpp
//万能引用 参数可接收左值或右值
template<typename T>
void Test1(T&& a)
{
}

//参数不能接收右值
template<typename T>
void Test1(T& a)
{
}
```

**不是万能引用的情况**：
```
1.template <typename T> void f(const T&& x); → 不是万能引用（const 修饰）。
2.类成员函数中的 T&&（T 非模板推导时）。
3.typedef 或 using 后的 T&&（T 已确定）。
```

# 引用折叠规则和发生引用折叠的情况
折叠的规则

| 组合情况     | 折叠结果  | 规则简述               |
| -------- | ----- | ------------------ |
| `T& &`   | `T&`  | 左值引用 + 左值引用 → 左值引用 |
| `T& &&`  | `T&`  | 左值引用 + 右值引用 → 左值引用 |
| `T&& &`  | `T&`  | 右值引用 + 左值引用 → 左值引用 |
| `T&& &&` | `T&&` | 右值引用 + 右值引用 → 右值引用 |

发生引用折叠的情况
1. 模板编程中的万能引用，当一个函数模板的参数被声明为“万能引用”（形式为 `T&&`）时，根据传入实参的值类别（左值或右值），模板参数 `T` 会被推导为不同的类型。如果传入的是左值，`T` 会被推导为左值引用类型（如 `Widget&`），此时函数签名中就会形成 `Widget& &&` 这样的“引用的引用”。编译器随后应用引用折叠规则，将其折叠为单一的左值引用 `Widget&`。如果传入的是右值，`T` 被推导为非引用类型（如 `Widget`），则 `T&&` 直接就是右值引用 `Widget&&`，无需折叠。
   如果传入的实参本身就是一个右值引用类型，那么模板参数 `T` 就会被推导为右值引用类型（如 `Widget&&`）。此时，函数签名中就会形成 `Widget&& &&` 这样的“引用的引用”。编译器随后应用引用折叠规则，将其折叠为单一的右值引用 `Widget&&`。例如，在调用 `func(std::move(obj))` 时，如果 `func` 是模板函数 `template<typename T> void func(T&& param)`，那么 `T` 被推导为 `Widget`，`T&&` 即为 `Widget&&`，无需折叠；但如果显式指定了模板参数为右值引用类型，如 `func<Widget&&>(std::move(obj))`，则会发生 `Widget&& &&` 到 `Widget&&` 的折叠。
2. auto推导变量类型，当使用 `auto&&` 声明变量时，它同样构成了一个“万能引用”。根据初始化表达式的值类别，`auto` 会被推导为不同的类型，如果初始化表达式是左值，就会产生引用的引用，进而触发折叠。例如，`int x; auto&& rx = x;` 中，`auto` 被推导为 `int&`，`auto&&` 即 `int& &&`，折叠后 `rx` 的类型为 `int&`。
3. 在定义类型别名时，如果被别名的类型本身是引用类型，那么引用折叠规则同样适用。例如：
```cpp
typedef int& T;       // T 是 int&
typedef T& TR;        // TR 是 int& &，折叠后为 int&
```
在这个例子中，`TR` 的实际类型是 `int&`，而非“引用的引用”。这允许开发者定义复杂的类型别名而无需担心产生非法的类型。
4. 当 `decltype` 作用于一个本身是引用类型的表达式时，其返回的类型会包含引用。如果这个结果被用于另一个需要引用类型的上下文（例如，作为函数返回类型的一部分），就可能产生引用的引用，从而触发折叠。例如，在函数 `auto func(int& a) -> const decltype(a)&;` 中，`decltype(a)` 得到 `int&`，与尾随的 `&` 结合形成 `int& &`，经引用折叠后，函数的实际返回类型为 `int&`。
