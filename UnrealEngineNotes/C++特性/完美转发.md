# 前情提要
在了解[[左值与右值和万能引用，引用折叠]]后
```cpp
void Test(int& a)
{
	cout << "Left value" << endl;
}

void Test(int&& a)
{
	cout << "Right value" << endl;
}

template<typename T>
void Test1(T&& a)
{
	Test(a);
}

int main()
{
	int a = 1;
	Test1(a);
	Test1(1);
	return 0;
}
```

输出：
Left value
Left value

# 为什么会诞生完美转发？
我们在使用万能引用的时候会遇到一个问题，无论T被推导成int& 还是 int（针对上述两种情况）
最后我们调用的都是左值引用的Test(int& a); 。无论是左值引用（`T&`）还是右值引用（`T&&`），一旦被声明并初始化，**这个引用变量本身就是一个左值**。也就是说当我们传入的是右值的时候，这个右值的性质丢失了，在内部使用的时候参数变成左值了。

# 使用完美转发
使用std::forward< T >();
```cpp
template<typename T>
void Test1(T&& a)
{
	Test(forward<T>(a));
}
```
传入左值T的推导类型是int&，传入右值T的推导类型是int。
随后a就会继续抱持左值或右值的属性进行传递。
如果不能理解可以看下std::forward的底层代码
```cpp
template constexpr T&& forward(typename std::remove_reference::type& t) noexcept 
{ 
	return static_cast(t); // 关键就在这个 static_cast 
}
```

它利用**引用折叠规则**，根据**原始推导出的 T** 来决定 cast 成什么：
```
1.原始传入的是右值 → T 被推导为 int（非引用） std::forward<int>(u) → static_cast<int&&>(u) → 产生纯右值（rvalue），触发移动！

2.原始传入的是左值 → T 被推导为 int& std::forward<int&>(u) → static_cast<int& &&>(u) → 折叠成 int&（左值引用），保持拷贝语义。

一句话总结：std::forward<T>(u) 把 u 强制转换回它原本传入时的值类别（左值就左值，右值就右值）。
```
